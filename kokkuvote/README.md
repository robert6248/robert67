# Kokkuvõte

Git on versioonihaldussüsteem.  
Seda kasutatakse failide ja muudatuste jälgimiseks.  
Git aitab näha, kes mida muutis ja millal muutis.  
GitHub on veebikeskkond, kus saab Git repositooriume hoida.  

Commit tähendab salvestatud muudatust.  
Commiti abil saab hiljem vaadata töö ajalugu.  
Push tähendab, et muudatused saadetakse GitHubi.  
Commit tehakse kõigepealt enda projektis ja push saadab selle internetti.  

Branch ehk haru on eraldi tööversioon.  
Harud on kasulikud, sest seal saab teha muudatusi ilma põhitööd rikkumata.  
Pull Request on ettepanek muudatused põhihaarde lisada.  
Selle abil saab enne kontrollida, mida muudeti.  

Git on parem kui failide kopeerimine USB-le, sest Git hoiab ajalugu alles.  
USB peal võivad failid segamini minna või kaduda.  
GitHubis saab kasutada ka Issues funktsiooni, mis töötab nagu ülesannete nimekiri.  
See seostub projektijuhtimisega, sest ülesandeid saab planeerida ja jälgida.  

Ma arvan, et GitHubi saab edaspidi kasutada koolitööde ja IT projektide jaoks.  
Kõige lihtsam oli failide loomine GitHubis.  
Kõige raskem oli aru saada branchidest ja Pull Requestist.
